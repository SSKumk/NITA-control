В файлах данных допускаются пустые строки и комментарии - строки, начинающиеся с решетки #
Такие строки игнорируются при чтении

--------------------------------------------------------------------------------------------------

Файл с точками
~~~~~~~~~~~~~~

В первой строке: Единицы измерения для каждой позиции (m, km, NM, m_s, km_h, NM_h)
 
Со второй перечисляются все точки (каждая следующая с новой строки) в формате:  
Название x y z Vmin Vmax флаг_посадочной_полосы 
Название начинается с буквы.
    x,y,z из R;
    Vmin,Vmax из R+;
    флаг_посадочной_полосы - LAND иначе пусто или ноль.

Пример:
m km NM km_h NM_h
P1 0 0 0 10 20 LAND
P2 1 1 1 10 22 0
P3 3 3 3 10 24

Файл со схемами
~~~~~~~~~~~~~~~

 - Каждая схема в одной строке
 - Схемы соединяются только через конечные и начальные точки (кроме зоны ожидания)
 - В начале строки, описывающей схем может стоять знак "+"; его наличие говорит, что заход на такую схему требует 1 взаимодействия (для описания в точках "развилок", которые являются начальными для нескольких схем)
 
 - Линейный участок - перечисление точек через пробел; номинальная траектория - проход через все точки, вариативности нет, 0 взаимодействий; конечная точка одна - последняя
 - Зона ожидания - имя точки: мин_время макс_время, время = <число><единица измерения>; номинальная траектория - пройти мимо ЗО, заход на ЗО - 1 взаимодействие
 - Сегмент спрямления - [тип]StrFrom(перечисление точек)StrTo(перечисление точек)
   - только последняя из точек StrFrom может быть быть в StrTo
   - конечные точки - все в StrTo и последняя в StrFrom (если она не в StrTo)
   - Участок спрямления - тип = S; номинальная траектория - через все точки StrFrom, сход с нее - 1 взаимодействие
   - Веер - тип = F, только одна точка в StrTo, отсутвует в StrFrom; номинальная траектория - заход на веер, прямой проход - 1 взаимодействие, сход с веера - 1 взаимодейстие; сход обязателен!
   - Тромбон - тип = T, только одна точка в StrTo; номинальная траектория - прямой проход мимо тромбона; заход на тромбон - 1 взаимодействие, сход с тромбна - 1 взаимодействие; в случае отсутсвие взаимодействия на сход тромбона ВС проходит весь тромбон
  
В приницпе, можно убрать "тип" и собирать каждую из схем со спрямлением из отдельных элементов. Но "тип" удобен для задания весовых коэффициентов взаимодействий диспетчер-пилот на каждом виде схем
